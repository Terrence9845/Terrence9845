# WiFi Connectivity Troubleshooting

## IT Support Case Study #1




Issue: User Unable to Connect to WiFi

Problem description
A user reports that their laptop is unable to connect to the office WiFi network. Other colleagues appear to be connected without any issues, suggesting the problem may be related to the user’s device rather than the network itself.



2. Initial Questions Asked

When handling a support request, it is important to gather key information before beginning troubleshooting
Are other devices able to connect to the WiFi?
This helps determine whether the issue is with the user’s device or with the router/network itself.


Are you receiving any specific error message?
Error messages can provide valuable clues about the cause of the problem and may indicate whether the issue is related to authentication, network configuration, or hardware.

When did the issue start?
Understanding when the problem began can help identify recent changes that may have caused the issue, such as software updates or network configuration changes.

Have you recently changed your password?
If the network password was recently updated, the device may still be trying to connect using an outdated credential.



Are you working remotely or in the office?
Location can affect connectivity. For example, weak signal strength or environmental obstacles can interfere with WiFi access.


Is WiFi turned on in your device settings?


Sometimes WiFi may be accidentally disabled, or Airplane Mode may be enabled.

3. Step-by-Step Troubleshooting Process
Step 1: Check WiFi is enabled
Confirm that WiFi is enabled on the device and that Airplane Mode is turned off. If WiFi is disabled, enable it and attempt to reconnect to the network.

Ensure Airplane Mode is disabled. Yes this is one of the most overlooked options as people don’t even realise that they have this on at times. When airplane mode is activated it blocks all other entities which makes wifi harder to access.


Step 2: Restart the device
Restarting the laptop can resolve temporary network or software glitches that may be preventing the device from connecting properly.
Step 3: Forget and Reconnect to Network
Open Network Settings
Select the current WiFi network


Click Forget


Reconnect to the network and re-enter the correct password
This step helps remove any incorrect or outdated saved network credentials.

4. Tools Used

Windows Network Settings


Command Prompt (ipconfig)


Device Manager
5. Root Cause
The issue was caused by the device storing an outdated WiFi password after the network credentials had been changed.
6. Resolution

The network was forgotten and reconnected using the updated password. The device successfully connected to the WiFi network.
7. Reflection
This case highlights the importance of gathering information before troubleshooting. Many connectivity issues are caused by simple configuration problems rather than hardware failures.
